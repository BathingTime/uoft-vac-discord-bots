# Frodo Meet
---

### Meeting Entries
Data for meeting plans will be stored in a txt file in entries.

Each entry consists of 4 lines:
1. Date: `YYYY, MM DD, HH:MM`.
2. Title.
3. Description.
4. Participants: list of pings, separated by spaces.
    - For members, use `<@…>`, where … is the member's <u>user ID</u>.
    - For roles, use `<@&…>`, where … is the <u>role ID</u>.
    - For everyone / here, use @everyone / @here respectively.

Entries are stored in chronological order based on date.
